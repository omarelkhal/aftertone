Hear a short line after your coding agent answers

On-device Supertonic ONNX through a tiny local HTTP daemon — models stay loaded, hooks stay fast, audio stays on your machine.

Today

What you get

Cursor integration

afterAgentResponse can trigger optional TTS from reply text. Prefer a <spoken_summary> line written for speech.

Daemon + prepare

speak_summary_prepare.py builds JSON for POST /say; tts_daemon.py serves localhost and plays audio.

Bootstrap

bash scripts/bootstrap.sh runs uv sync and pulls ONNX assets when assets/onnx/ is missing.

Install

git clone https://github.com/omarelkhal/aftertone.git
cd aftertone
bash scripts/bootstrap.sh

Open the repo as your Cursor workspace root so project hooks load. See the README for daemon control, smoke tests, and diagnostics.
