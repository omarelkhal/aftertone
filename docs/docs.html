Getting started

You need git, uv (or --install-uv on the installer), ONNX weights under assets/, and the TTS daemon running before Cursor can speak. Change settings with slash commands in Agent chat — not by editing TOML by hand.

  1. One-line install (recommended)

    Clones to ~/aftertone by default, runs uv sync, and downloads Supertonic models.

    Terminal
    curl -fsSL https://raw.githubusercontent.com/omarelkhal/aftertone/main/scripts/install.sh | bash -s -- --install-uv --start-daemon

    Manual clone: git clone … && bash scripts/bootstrap.sh. Add hooks to another repo: bash -s -- --into .. See scripts/README.md.

  2. Start the TTS daemon

    The daemon loads ONNX models once and serves POST /say on localhost. Keep it running while you code.

    Terminal
    cd py
uv run python tts_daemon_ctl.py start --repo-root ..

    Check health: uv run python tts_daemon_ctl.py status --repo-root ..

  3. Smoke-test without Cursor

    Confirms assets, audio output, and the prepare → daemon path.

    From repo root
    bash py/test_speak_summary_pipeline.sh

    You should see OK: and hear a short test phrase.

  4. Open Aftertone in Cursor

    Use the repository root as the workspace folder so project hooks and .cursor/hooks.json load.

  5. Trigger a real agent reply

    Run /aftertone-on, ask something substantive, and listen for one short line after the reply. If not, see Troubleshooting.

Slash commands (Cursor)

Open the Aftertone folder as the workspace root. In Agent chat, type / and pick an aftertone- command. The agent runs the repo scripts for you — do not hand-edit speak_summary.toml for everyday changes.

CommandWhat it changesDaemon restart?
/aftertone-toggleFlip spoken TTS on/offNo
/aftertone-on · /aftertone-offForce on or offNo
/aftertone-statusShow settings + daemon healthNo
/aftertone-langLanguage (syncs agent spoken-summary rule)No
/aftertone-speedPlayback speedNo
/aftertone-modequeue or interruptNo
/aftertone-voiceVoice picker (human names, e.g. Sara (female), James (male))Yes (command restarts for you)
Voices Pickers show names like Elena (female) and James (male); TOML still stores preset ids (F1–F5, M1–M5). You can pass a first name to the config CLI (e.g. Sara → F4).

Command definitions: .cursor/commands/ on GitHub.

Cursor setup

Hooks enabled

Cursor Settings → Hooks should show no errors for this project’s .cursor/hooks.json.

Trusted workspace

Project hooks do not run in restricted mode. Trust the folder when Cursor asks.

version: 1

.cursor/hooks.json must include "version": 1 (numeric) or hooks fail to load.

What runs: afterAgentResponse → speak_summary.sh → speak_summary_prepare.py → POST /say on the daemon. The hook uses inline reply text from the payload (not the transcript file).

Verify Cursor called the hook After one agent reply: bash py/diagnose_speak_hooks.sh or inspect .cursor/hooks/state/hook_payload_trace.jsonl for afterAgentResponse with inline_after_response_ok: true.

Daemon commands

Run from py/ with --repo-root .. (one level up to the repo root):

cd py
uv run python tts_daemon_ctl.py start --repo-root ..
uv run python tts_daemon_ctl.py status --repo-root ..
uv run python tts_daemon_ctl.py restart --repo-root ..
uv run python tts_daemon_ctl.py stop --repo-root ..
Change in TOMLNeed restart?
speed, lang, total_step, mode, quiet_hours, text limits, heuristicsNo — read each hook run
port, onnx_dir, voice_*, use_gpuYes — restart
Port mismatch While the daemon runs, the hook posts to the port in .cursor/hooks/state/tts-daemon.port. If you change port in TOML without restarting, speech may go to the wrong port. Run restart after changing port.

Configuration

Use slash commands for enabled, lang, speed, mode, and voice_type. The file .cursor/hooks/speak_summary.toml is updated by those commands; edit it directly only for advanced keys. Common knobs:

KeyWhat it does
enabledMaster switch — false silences all speech
langONNX language and the language agents should use in <spoken_summary> (no auto-translation)
voice_type / voice_stylePreset id (e.g. F4) or path to voice JSON; pick via /aftertone-voice
speedPlayback speed (1.0 = baseline)
quiet_hourse.g. 22:00-08:00 — no speech in that window (local clock)
spoken_summary_max_charsCap for tag text (default 360)
only_speak_spoken_summarytrue = never use heuristic fallback

/aftertone-lang updates lang and syncs the Cursor rule automatically. Manual sync from repo root: uv run --directory py python sync_spoken_rule_lang.py

Spoken summaries

Best quality comes from a deliberate tag at the end of the agent message:

<spoken_summary>
Short line written for listening, in the same language as lang in speak_summary.toml.
</spoken_summary>

Without the tag, prepare may use one or two heuristic sentences (shorter when the reply is mostly code fences). Set only_speak_spoken_summary = true if you never want English (or other) fallback when lang is not English.

Rules for agents live in .cursor/rules/spoken-summary.mdc.

Troubleshooting

Nothing speaks

Run bash py/test_speak_summary_pipeline.sh. Check daemon status, Cursor Hooks panel, trusted workspace, and tail .cursor/hooks/state/speak_summary-hook.log.

Wrong language or accent

Match lang in TOML and write <spoken_summary> in that language. Run sync_spoken_rule_lang.py after changing lang.

TOML change had no effect

Voice, port, GPU, and ONNX path need tts_daemon_ctl.py restart. Speed and lang apply per request without restart.

Quiet hours

Clear quiet_hours or test with SPEAK_SUMMARY_IGNORE_QUIET=1 in the environment.
